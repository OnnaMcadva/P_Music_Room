## **Music Room**

---

### О чем проект (кратко)
Создание мобильного приложения для совместного прослушивания музыки. 
**Главные фичи (минимум 2 из 3):**
1. **Music Track Vote:** Голосование за треки в плейлисте (с учетом геолокации/времени).
2. **Music Playlist Editor:** Совместное редактирование плейлиста в реальном времени.
3. **Music Control Delegation:** Передача управления музыкой друзьям.

**Технические требования:** Авторизация (Google/FB), REST API (JSON, Swagger), логирование, безопасность, нагрузочное тестирование, CI/CD.

---

### Решение по бэкенду (Python)
Выбор Python — **отличный вариант**. ТЗ не ограничивает стек бэкенда. 
* **Рекомендация:** Использовать **FastAPI**. Он работает быстрее Django, а главное — **автоматически генерирует документацию Swagger**, которая строго обязательна по ТЗ.
* Для работы в реальном времени (совместный плейлист) понадобятся **WebSockets** (в FastAPI они поддерживаются из коробки).

---

### Распределение ролей на 4 человек (с учетом бонусов)

Чтобы получить бонус за мультиплатформенность, фронтендерам лучше писать мобилку на **Flutter** или **React Native** (один код для iOS, Android и Web).

#### **Участник 1: Backend-разработчик (тот, кто хочет Python)**
* **Зона ответственности:** Архитектура БД, API, Авторизация и Безопасность.
* **Задачи:**
  * Разработка базы данных (PostgreSQL/Redis).
  * Реализация авторизации (Email/Пароль + OAuth Google/Facebook).
  * Создание REST API (FastAPI + авто-Swagger).
  * Логирование всех действий пользователей на сервере.
  * *Бонус:* Реализация логики платной/бесплатной подписки на уровне API.

#### **Участник 2: DevOps / Real-time Backend (Помощник бэкендера)**
* **Зона ответственности:** Реал-тайм фичи, Нагрузочное тестирование, CI/CD.
* **Задачи:**
  * Реализация WebSockets для мгновенного обновления очереди треков (для *Playlist Editor* и *Track Vote*).
  * Нагрузочное тестирование сервера (инструментом **Locust** — он как раз на Python).
  * Настройка CI/CD, Docker и деплой сервера.
  * Решение проблемы конкуренции (что делать, если два юзера одновременно голосуют за один трек).

#### **Участник 3: Frontend Lead (Мобильное приложение)**
* **Зона ответственности:** UI/UX, Авторизация на клиенте, Профиль.
* **Задачи:**
  * Создание каркаса приложения (Flutter или React Native).
  * Интеграция SDK для входа через Google/Facebook.
  * Экран профиля (настройки приватности, музыкальные предпочтения).
  * *Бонус (Мультиплатформенность):* Адаптация интерфейса под Web-версию.

#### **Участник 4: Frontend Features & IoT (Интеграция фич и Бонусы)**
* **Зона ответственности:** Логика сервисов на клиенте, Офлайн-режим, IoT.
* **Задачи:**
  * Интеграция API для голосования за треки и редактирования плейлистов.
  * *Бонус (Офлайн-режим):* Локальное сохранение данных и синхронизация при появлении сети (разрешение конфликтов).
  * *Бонус (IoT):* Интеграция iBeacon ( push-уведомления при приближении к гео-событию).
